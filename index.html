Binary Search Tree

一棵二叉查找树(BST)是一棵二叉树，其中每个结点都含有一个Comparable的键(以及相关联的值)且每个结点的键都大于其左子树中的任意结点的键而小于右子树的任意结点的键

简单来说就是：
根结点的key比左子树的都大，比右子树的都小
或者
左子树的key都比根结点小，右子树的key都比根结点大

BST Height

一棵二叉树的高度是其根结点(root) 至其 最深的结点(the deepest node) 的路径长度.

△一棵只有根结点的树，其高度为0

BST Height Algorithm
BST高度计算算法 (pesudo code)

Input: Binary Tree T
Output: Height of T

if T = ∅   then
   return -1
else
   return max(BSTHeight($T_L$), BSTHeight($T_R$)) + 1
end if

Pre-order

在当前子树的左右子树被遍历之前，先visit其根结点(root)。
既每个结点按照以下优先级进行操作。
visit -> left -> right

output: 16 -> 6 -> 4 -> 1 -> 5 -> 12 -> 26 -> 20 -> 28 -> 27

In-order


left -> visit -> right

output: 1 -> 4 -> 5 -> 6 -> 12 -> 16 -> 20 -> 26 -> 27 -> 28

Post-order

left -> right -> visit

output: 1 -> 5 -> 4 -> 12 -> 6 -> 20 -> 27 -> 28 -> 26 -> 16
